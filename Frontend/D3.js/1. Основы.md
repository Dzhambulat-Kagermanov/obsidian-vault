
### Dataset:

```ts
const dataset = [ 5, 10, 15, 20, 25 ];

d3.select("body").selectAll("p")
.data(dataset)
.enter()
.append("p")
.text("New paragraph!");
```

1. _d3.select("body")_ - находит _body_ элемент в DOM'e, и передает ссылку на него следующему методу в цепочке.
2. _.selectAll("p")_ - выбирает все параграфы, расположенные в _body_. Так как они еще не существуют, то в выборке ничего нет. Эта пустая выборка должна представлять собой набор параграфов; они будут вскоре созданы.
3. _.data(dataset)_ - считает количество значений и анализирует их. В нашем наборе 5 значений, так что все методы, вызываемые после этого, будут выполняться пять раз, то есть по разу на каждое значение.
4. _.enter()_ - Чтобы создать новый, привязанный к данным элемент, мы используем метод _enter()_. Сначала метод смотрит на DOM, а потом на данные, переданные ему. Если данных больше, чем элементов, значит _enter()_ создает новый промежуточный элемент. Дальше он возвращает ссылку на этот промежуточный элемент, после чего вызывается следующий метод.
5. _.append("p")_ - берет промежуточный элемент, созданный с помощью метода _enter()_, и добавляет _p_-элемент вместо него в DOM. Ура, товарищи! Дальше метод возвращает ссылку на уже нормальный вновь созданный элемент, и мы переходим к следующему шагу в нашей цепочке.
6. _.text("New paragraph!")_ - берет ссылку на вновь созданный _p_-элемент и вставляет в него текстовое значение.

D3 работает по принципу сравнения двух массивов:
1. Массив **DOM-элементов**, которые сейчас есть на странице (результат `selectAll`).
2. Массив **Данных**, которые ты передаешь через `.data(dataset)`.

После вызова `.data(dataset)`, D3 внутренне делит элементы на три группы:

| Группа     | Условие                      | Что делать?                | Метод                  |
| ---------- | ---------------------------- | -------------------------- | ---------------------- |
| **Enter**  | Данных больше, чем элементов | **Создать** новые элементы | `.enter()`             |
| **Update** | Данные и элементы совпадают  | **Обновить** существующие  | (цепочка продолжается) |
| **Exit**   | Элементов больше, чем данных | **Удалить** лишние         | `.exit()`              |

**Пример кода с разбором:**
```ts
// Шаг 1: Выбираем ВСЕ круги (даже если их 0).
// Результат: Пустая группа или группа существующих кругов.
svg.selectAll("circle")

// Шаг 2: Привязываем данные.
// D3 сопоставляет: [Круг1, Круг2...] <-> [Data1, Data2...]
// Так как кругов нет, все данные попадают в группу "Enter".
.data(dataset)
  
// Шаг 3: Говорим D3: "Работай только с теми данными,
// для которых НЕ хватило элементов".
// Это виртуальные placeholder-ы для новых данных.
.enter()                 

// Шаг 4: Для каждого placeholder из .enter()
// создаем реальный DOM-элемент <circle>.
.append("circle")
```

**Наглядный пример с несколькими запусками одного и того же кода**:
* **Запуск 1 (SVG пустой):**
	Данные по умолчанию: `dataset = [10, 20, 30]`
	1. `selectAll("circle")` → находит 0 кругов.
	2. `.data([10, 20, 30])` → видит: данных 3, кругов 0.
	3. `.enter()` → создает 3 виртуальных места.
	4. `.append("circle")` → рисует **3 новых круга**.
* **Запуск 2 (Данные изменились на `[10, 20, 30, 40, 50]`):**
	1. `selectAll("circle")` → находит 3 существующих круга.
	2. `.data([10, 20, 30, 40, 50])` → видит: данных 5, кругов 3.
	3. `.enter()` → создает 2 виртуальных места (для 40 и 50).
	4. `.append("circle")` → рисует **только 2 новых круга**. Старые остаются на месте (попадают в группу Update, но в твоем коде нет инструкций для обновления, поэтому они просто висят).
* **Запуск 3 (Данные уменьшились до `[10]`):**
	1. `selectAll("circle")` → находит 5 кругов.
	2. `.data([10])` → видит: данных 1, кругов 5.
	3. `.enter()` → пусто (новых создавать не надо).
	4. `.append("circle")` → ничего не делает.
	5. _(Важно!)_ Чтобы удалить лишние, нужно добавить цепочку:
```ts
svg.selectAll("circle")
  .data(newDataset)
  .exit()        // Выбирает лишние элементы (4 круга)
  .remove();     // Удаляет их из DOM
```

Начиная с D3 v6, появилась более короткая запись, которая делает то же самое автоматически (создает, обновляет и удаляет):
```ts
svg.selectAll("circle")
  .data(dataset)
  .join("circle") // Автоматически делает enter().append() + update + exit().remove()
  .attr("r", d => d); 
```

### Установка атрибутов

Метод _attr()_ используется для установки элементу атрибута и его значения. HTML-атрибутом является любая пара свойство/значение, которые можно включать для элемента между скобками _<>_.  Для установки класса _bar_ мы можем использовать:
```ts
.attr("class", "bar")

// Или с помощью classed
.classed("bar", true)
```

 Внутрь `.text(...)`, можно передать callback функцию которая принимает первым параметром текущие данные, а вторым index рендера для промежуточного элемента. Например:
```ts
.text(function(d, i) {
	console.log(i)
	return "I can count up to " + d;
});
```

Еще пример с style:
```ts
.style("height", function(d, i) {
	return d * i * 5 + "px";
});
```
