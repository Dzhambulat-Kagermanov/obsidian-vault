**Масштабирование (Нормализация для отображения)** — это создание функции, которая говорит - если значение данных равно **X**, то на экране оно должно находиться в точке **Y**. Масштабирование нужно чтобы переводить значения в нужный диапазон значений для адаптивного отображения данных.

**Пример линейного масштабирования:**
```ts
const scale = d3.scale.linear();

console.log(scale(2.5)) // 2.5
```
Так как у нас нет еще входной области значений и выходного диапазона значений, эта функция масштабирует входные значения к выходным в пропорции 1:1

Мы можем задать входную область значений 100-500, передавая эти значения в метод _domain()_ в качестве массива:
```ts
scale.domain([100, 500]);
```

Также мы устанавливаем выходной диапазон вызовом метода _range()_:
```ts
scale.range([10, 350]);
```

**Результат:**
```ts
scale(100); // Returns 10
scale(300); // Returns 180
scale(500); // Returns 350
```

### Точечная диаграмма с масштабированием:

```ts
const dataset = [
	[5, 20], [480, 90], [250, 50], [100, 33], [330, 95],
	[410, 12], [475, 44], [25, 67], [85, 21], [220, 88]
]

const w = 500
const h = 50

const svg = d3.select("body")
.append("svg")
.attr("width", w)
.attr("height", h)

svg.selectAll("circle")
.data(dataset)
.enter()
.append("circle")
.attr("cx", function(d) {
	return d[0]
})
.attr("cy", function(d) {
	return d[1]
})
.attr("r", function(d) {
	return Math.sqrt(h - d[1])
});

svg.selectAll("text")
.data(dataset)
.enter()
.append("text")
.text(function(d) {
	return d[0] + "," + d[1]
})
.attr("x", function(d) {
	return d[0]
})
.attr("y", function(d) {
	return d[1]
})
.attr("font-family", "sans-serif")
.attr("font-size", "11px")
.attr("fill", "red")
```

Вместо того, чтобы указывать фиксированные значения для задания области входных значений, мы можем использовать подходящие для этих целей методы _min()_ и _max()_, чтобы анализировать наш набор данных автоматически. Например, код ниже перебирает каждое значение _x_ в наших подмассивах и возвращает наибольшее значение:
```ts
d3.max(dataset, function(d) { //Returns 480
	return d[0] //References first value in each sub-array
});
```

Соберем наш код вместе и получим масштабирующую функцию для оси _x_ и _y_:
```ts
const xScale = d3.scale.linear()
.domain([0, d3.max(dataset, function(d) { return d[0] })])
.range([0, w])

const yScale = d3.scale.linear()
.domain([0, d3.max(dataset, function(d) { return d[1] })])
.range([0, h])
```

Заменяем аттрибуты для кругов на масштабируемые значения:
```ts
.attr("cx", function(d) {
	return xScale(d[0])
})
.attr("cy", function(d) {
	return yScale(d[1])
})
```

### Padding-и для кругов:

Так как некоторые элементы выходят за пределы можно добавить padding-и:
```ts
const padding = 20
```

Теперь можно заменить `.range([0, w])` и `.range([h, 0])`  на:
```ts
.range([padding, w - padding]);

.range([h - padding, padding]);
```

### Методы d3.scale.linear():

_d3.scale.linear()_ имеет несколько других удобных методов:
- **_nice()_** - он говорит масштабирующей функции принять все что угодно в качестве выходного диапазона и расширить этот диапазон в обе стороны до ближайшего округленного значения. Выдержка из wiki D3: "Например, для области входных данных \[0.20147987687960267, 0.996679553296417\], метод вернет \[0.2, 1\]". Это удобно для людей, кому сложно читать цифры, подобные этой: 0.20147987687960267.
- **_rangeRound()_** - используйте этот метод вместе _range()_, и все выходные значения масштабирующей функции будут округлены до ближайшего целого числа. Этот метод полезен случае, когда вам нужны фигуры с точными значениями пикселей, чтобы избежать нечетких краев, которые могут возникнуть при сглаживании.
- **_clamp()_** - по-умолчанию, линейное масштабирование может возвращать значения вне указанного выходного диапазона. Например, если дано значение, находящееся за пределами входной области значений, масштабирующая функция также вернет значение, расположенное вне диапазоны выходных значений. Если вызвать метод _.clamp(true)_ в масштабирующей функции, заставляет все возвращаемые значения придерживаться выходного диапазона. Это означает, что такие выходные значения будут приравнены к минимальному или максимальному(что ближе) значению выходного диапазона.

### d3.scalePow:

Функция d3.scalePow() возвращает определенную степень числа:

```ts
const data =[1, 2, 3, 4, 5];

const pow = d3.scalePow().exponent(2);

for (let i=0; i<data.length; i++) {
	data[i]=pow(data[i]);
}

console.log(data); // [ 1, 4, 9, 16, 25 ]
```

Можно использовать интервалы, в этом случае формула будет также меняться:

```ts
const data =[1, 2, 3, 4, 5];

const pow = d3.scalePow().exponent(2).domain([1, 10]).rangeRound([1, 10]);

for (let i=0; i<data.length; i++)
	data[i]=pow(data[i]);
}
console.log(data); // [ 1, 1, 2, 2, 3 ]
```

### d3.scaleTime:

Позволяет сопоставлять линейные интервалы с временными отрезками, то может быть полезно, например, если мы решим вывести по одной оси даты:

```ts
const data = [ // интервал дат
    new Date(2025, 0, 10),
    new Date(2025, 0, 20),
    new Date(2025, 0, 31),
    new Date(2025, 1, 8),
    new Date(2025, 1, 16)
];

const time = d3.scaleTime()
	.domain(
		[
			new Date(2025, 0, 1),
			new Date(2025, 2, 1)
		]
	)
	.rangeRound([0, 300]);

for (let i=0; i<data.length; i++) {
	data[i]=time(data[i]);
}

console.log(data); // [ 46, 97, 153, 193, 234 ]
```

Здесь некоторый набор дат из массива data интерполируется на отрезок от 0 до 300.

### d3.scaleBand

Еще один тип функций масштабирования, который часто применяется - d3.scaleBand(): они принимают дискретные данные, которые необязательно представляют количественные значения, и по ним генерируют количественные результаты. Эта функция удобна, когда надо расположить данные, которые нельзя выразить количественно

```ts
const data = [
    {id:1, lang: "Python", rate: 23.85},
    {id:2, lang: "C++", rate: 11.08},
    {id:3, lang: "Java", rate: 10.36},
    {id:4, lang: "JavaScript", rate: 3.46},
    {id:5, lang: "Go", rate: 2.78},
]

const yScale = d3.scaleBand()
	.domain(data.map(d => d.lang))
	.range([0, 700]);

data.forEach(d=> console.log(d.lang, ":", yScale(d.lang)));
```

Здесь мы вызываем функцию `d3.scaleBand()` и сохраняем полученную функцию в константу yScale (она отвечает за распределение элементов по оси Y). Функция `domain` принимает массив, содержащий все названия языков из нашего набора данных. Мы генерируем этот массив с помощью стандартной функции JavaScript map(). А функция `range()` охватывает пространство от 0 пикселей до 700. Для проверки значений для каждого языка выводим результат функции yScale на консоль:

```log
Python : 0
C++ : 140
Java : 280
JavaScript : 420
Go : 560
```

Здесь мы видим, что для каждого языка генерируется число от 0 до 700. Впоследствии мы можем использовать это для расположения прямоугольников гистограммы по оси Y.

Создаваемый объект-функция предоставляет удобный метод -_bandwidth()_, который возвращает толщину полос, которая пропорциональна количеству полос и доступному пространству. Например:

```ts
`const data = [`

        `{id:1, lang: "Python", rate: 23.85},`

        `{id:2, lang: "C++", rate: 11.08},`

        `{id:3, lang: "Java", rate: 10.36},`

        `{id:4, lang: "JavaScript", rate: 3.46},`

        `{id:5, lang: "Go", rate: 2.78},`

    `]`

    `const koef = 10; // коэффициент масштабирования ширины`

    `const bottom = 250; // низ гистограммы`

    `const left = 10;  // отступ слева`

    `// функция для масштабирования по оси Y`

    `const yScale = d3.scaleBand()`

      `.domain(data.map(d => d.lang))`

      `.range([0, bottom])`

      .paddingInner(0.2);   // отступ между полосами

// функция для масштабирования по оси X
const  xScale = d3.scaleLinear()
	.domain([d3.min(data, d=>d.rate), d3.max(data, d=>d.rate)])
	.range([d3.min(data, d=>d.rate)*koef, d3.max(data, d=>d.rate)*koef]);

d3.select("svg")
	.attr("viewBox", `0 0 500 ${bottom}`)
	.selectAll("rect")
	.data(data)
	.join("rect")
	.attr("y", d => yScale(d.lang))
	.attr("height", yScale.bandwidth())
	.attr("width", d => xScale(d.rate))
	.attr("x", left)
	.attr("fill", "steelblue");
```
