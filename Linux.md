##### chmod (change mode) - изменить права доступа к файлу для пользователей:
* r w x === 4 2 1
	* rw- === 4+2
	* r-x === 4+1
* специальный бит - это 4-ый бит в аргументе chmod который может указываться для следующих конструкций:
	* SUID - выполнять команды от имени владельца исполняемой команды;
		* `chmod 4777 MY_DIRECTORY`
	* SGID - ???;
		*  `chmod 2777 MY_DIRECTORY`
	* StickyBite - разрешение полезно для защиты файлов от случайного удаления в среде, где несколько пользователей имеют права на запись в один и тот же каталог. пользователь может удалять файлы, только если выполняется одно из следующих условий:
		* Пользователь является владельцем файла;
		- Пользователь является владельцем каталога, в котором находится файл.
		* `chmod 1777 MY_DIRECTORY`

#### chown (change owner) - изменить владельца файла:
* Сменить владельцев для всех файлов в директории:
	* `chown -R user MY_DIRECTORY`

#### Процессы:
* Список процессов пользователя - `ps -u {{user}}`;
* Динамическая информация о процессах - `top`;
* Смена приоритета для процесса - `renice -n {{ПРИОРИТЕТ}} -p {{pid}}`;
* Завершить процесс - `kill {{pid}}`;
* Перевод процесса в фоновый режим - `в конце команды нужно добавить &;`
* Список потоковых процессов - `jobs`;
* Перевод процесса на передний режим - `fg {{pid или %номер_процесса_в_jobs}}`;
* Остановка процесса - `зажать CTRL+z`;
* Запуск остановленного процесса - `bg {{pid или %номер_процесса_в_jobs}}`

#### Оболочка:
* Очистка истории - `history -c`

#### Вывод информации о портах:
`ss -tulpn`
* t - TCP;
* u - UDP;
* l - Прослушиваемые;
* p - Процесс (требует sudo);
* n - Числовые IP адреса без DNS;
* a - Все соединения;
