* `api-server` - приложение-посредник между всеми компонентами, а также не посредственно вход в кластер;
* `etcd` - база данных ключ значение, отказоустойчивое и кластеризуемое хранилище. Хранятся настройки кластера;
* `controller` - слушает `api-server` с помощью функции watch тем самым следит за появлением манифестов для взаимодействия с подами;
* `scheduler` - обращается в `api-server` и следит не появился ли запрос для создания пода. В зависимости от требований пода к ресурсам, доступных ресурсов и прочих факторов решает на какой кластер разместить под;
* `kubelet` - запускается на каждой ноде. Приложение которое запускает поды, слушая `api-server` и следя за манифестами подов где уже есть информация от `scheduler`. Kubelet обращается к системе контейнеризации на своей ноде, смотрит не запущен ли такой под, если нет то система контейнеризации создаёт нужный контейнер;
* `pod` - базовый элемент приложения в кластере kubernetes;
* `kubeproxy` - запускается на каждой ноде. Отвечает за сеть сервисов;
* Дополнительные компоненты:
	* * `DNS Server`;
	* * `CNI (Container network interface)` - драйвер сети pod-ов;
	* `CSI (Container storage interface);
	* Дополнительные контроллеры;

Все компоненты обращаются к api-server-у, а api-server обращается только к etcd;

etcd хранилища в разных нодах связаны между собой;
