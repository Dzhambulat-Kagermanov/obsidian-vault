### Службы openstack:

OpenStack — это модульная облачная платформа, где каждый компонент (служба) отвечает за свою конкретную задачу. Вместе они создают полноценную инфраструктуру как услугу (IaaS).

##### 1. Службы хранения образов и данных
*   **Glance (Image Service)**
    *   **Предназначение:** Хранение, управление и доставка образов виртуальных машин (VM images).
    *   **Как работает:** Когда вы создаете новую виртуальную машину, Nova обращается к Glance, чтобы скачать образ операционной системы (например, Ubuntu, CentOS, Windows). Glance не хранит сами данные файлов пользователя, только шаблоны дисков для запуска VM.
    *   **Аналог:** Каталог ISO-образов или шаблонов дисков.

*   **Swift (Object Storage)**
    *   **Предназначение:** Масштабируемое хранилище неструктурированных данных (объектов).
    *   **Как работает:** Хранит данные в виде объектов (файлов) внутри контейнеров. Идеально подходит для резервных копий, архивов, медиа-контента (фото, видео) и статических сайтов. Данные реплицируются across множество серверов для надежности.
    *   **Аналог:** Amazon S3. Это *не* блочное хранилище для установки ОС, а хранилище файлов.

*   **Cinder (Block Storage)**
    *   **Предназначение:** Предоставление постоянных блочных устройств хранения для виртуальных машин.
    *   **Как работает:** Позволяет создавать "виртуальные жесткие диски", которые можно подключать к запущенным инстансам. В отличие от эфемерного диска (который удаляется при удалении VM), том Cinder живет своей жизнью: его можно отключить от одной VM и подключить к другой, делать снапшоты и бэкапы.
    *   **Аналог:** Amazon EBS или физический HDD/SSD, который можно воткнуть в любой сервер.

##### 2. Службы управления вычислениями и "железом"
*   **Nova (Compute Service)**
    *   **Предназначение:** Ядро OpenStack. Управляет жизненным циклом виртуальных машин.
    *   **Как работает:** Отвечает за создание, планирование (на каком физическом хосте запустить VM), остановку и удаление инстансов. Она координирует работу гипервизоров (KVM, VMware, Hyper-V и др.).
    *   **Роль:** Это "дирижер", который говорит физическим серверам, какую VM запустить.

*   **Ironic (Bare Metal Provisioning)**
    *   **Предназначение:** Управление физическими серверами ("голое железо") так, как если бы они были виртуальными машинами.
    *   **Как работает:** Позволяет разворачивать ОС напрямую на физическом оборудовании, минуя слой виртуализации. Полезно для задач, требующих максимальной производительности или специфического оборудования (GPU, высокоскоростные сети), которое сложно пробросить через виртуализацию.
    *   **Отличие от Nova:** Nova управляет VM, Ironic управляет физическими серверами.

##### 3. Сетевые службы

*   **Neutron (Networking)**
    *   **Предназначение:** Управление сетями как услугой (SDN).
    *   **Как работает:** Позволяет пользователям создавать собственные изолированные сети, подсети, маршрутизаторы, шлюзы, балансировщики нагрузки и правила фаервола. Обеспечивает связь между VM, внешним миром и хранилищами.
    *   **Возможности:** Плавающие IP-адреса (Floating IPs), VLAN, VXLAN, безопасность групп (Security Groups).

##### 4. Службы идентификации и интерфейса

*   **Keystone (Identity Service)**
    *   **Предназначение:** Единая точка аутентификации и авторизации.
    *   **Как работает:** Управляет пользователями, проектами (тенантами), ролями и правами доступа. Выдает токены доступа, которые другие службы (Nova, Glance и т.д.) проверяют, прежде чем выполнить запрос. Без Keystone ни одна другая служба не заработает.
    *   **Аналог:** Active Directory или OAuth-сервер для всей облачной платформы.

*   **Horizon (Dashboard)**
    *   **Предназначение:** Веб-интерфейс для управления OpenStack.
    *   **Как работает:** Предоставляет графическую панель управления, через которую пользователи могут запускать VM, создавать диски и настраивать сети, не используя командную строку (CLI) или API напрямую.

##### 5. Дополнительные важные службы

*   **Heat (Orchestration)**
    *   **Предназначение:** Автоматизация развертывания инфраструктуры.
    *   **Как работает:** Позволяет описывать всю инфраструктуру (сеть + VM + диски) в виде шаблонов (YAML/JSON). По одному шаблону Heat может развернуть сложное приложение со всеми зависимостями.
    *   **Аналог:** Terraform (но встроенный в OpenStack) или AWS CloudFormation.

*   **Ceilometer / Telemetry (Telemetry)**
    *   **Предназначение:** Сбор метрик и биллинг.
    *   **Как работает:** Считает потребление ресурсов (сколько часов работала VM, сколько ГБ трафика прошло, сколько места занято). Эти данные используются для мониторинга и выставления счетов пользователям.

*   **Magnum (Container Orchestration)**
    *   **Предназначение:** Развертывание движков оркестрации контейнеров.
    *   **Как работает:** Позволяет быстро создавать кластеры Kubernetes, Docker Swarm или Apache Mesos поверх инфраструктуры OpenStack.

##### Краткая схема взаимодействия (пример запуска VM):

1.  Пользователь входит через **Horizon**, который спрашивает токен у **Keystone**.
2.  Пользователь просит создать VM с Ubuntu и диском 50 ГБ.
3.  **Nova** получает запрос, проверяет квоты и выбирает свободный физический хост.
4.  **Nova** просит **Cinder** создать том 50 ГБ.
5.  **Nova** просит **Glance** предоставить образ Ubuntu.
6.  **Nova** просит **Neutron** выделить IP-адрес и настроить порт в сети.
7.  **Nova** дает команду гипервизору запустить VM, подключив том от Cinder и образ от Glance к сети от Neutron.
8.  **Ceilometer** начинает считать время работы этой VM.

### Основные команды OpenStack CLI:

##### Команды для работы с Flavor (Шаблонами):

**Получить список flavor:**
```bash
openstack flavor list
    [--public | --private | --all]
    [--long]
```
*Флаги:*
 `--public` — показать только публичные шаблоны ВМ;
 `--private` — показать только приватные шаблоны ВМ;
`--all` — показать все шаблоны ВМ: публичные и приватные (по умолчанию);
`--long` — выводить дополнительные поля;


**Просмотр информации о flavor-е:**
```bash
openstack flavor show <ИМЯ_ИЛИ_ID_ШАБЛОНА>
```
