
**Prometheus** — это база данных временных рядов (Time Series Database — TSDB), которая собирает метрики (числовые данные с временной меткой) с ваших серверов, приложений и сетей, позволяет делать по ним запросы и настраивать алерты (уведомления), если что-то идет не так.

В отличие от многих старых систем (как Zabbix или Nagios), Prometheus работает по принципу **Pull (Вытягивание)**:

1. **Scraping (Скрейпинг):** Сервер Prometheus сам периодически опрашивает целевые объекты (targets) по HTTP, забирая у них метрики.
2. **Хранение:** Все данные хранятся локально на диске сервера Prometheus в виде временных рядов.
3. **Модель данных:** Данные — это не просто числа, а пары `Метрика + Метки (Labels)`.
    - Пример: `http_requests_total{method="POST", handler="/api", status="200"}`

#### Основные компоненты:

Экосистема Prometheus состоит из нескольких частей:

1. **Prometheus Server:** Главный компонент. Скачивает метрики, хранит их и обслуживает запросы.
2. **Exporters (Экспортеры):** Небольшие программы, которые транслируют метрики из сторонних систем в формат, понятный Prometheus.
    - _Node Exporter:_ Метрики железа (CPU, RAM, Disk) Linux/Windows серверов.
    - _MySQL Exporter:_ Метрики базы данных.
    - _Blackbox Exporter:_ Проверка доступности (ping, http запросы).
3. **Pushgateway:** Специальный сервис для короткоживущих задач (batch jobs), которые не могут быть "опрошены" сервером, поэтому они сами "пушат" (отправляют) метрики в шлюз перед завершением.
4. **Alertmanager:** Отвечает за обработку алертов. Он получает сигнал от Prometheus, дедуплицирует их, группирует и отправляет уведомления (в Slack, Telegram, Email, PagerDuty).
5. **Grafana:** (Важное уточнение) Это **не** часть Prometheus, но они почти всегда используются вместе. Prometheus хранит данные, а Grafana строит красивые графики и дашборды на основе этих данных.

#### Типы метрик:

1. **Counter (Счетчик):** Только растет (или сбрасывается в 0 при перезапуске).
    - _Пример:_ Количество запросов, количество ошибок.
2. **Gauge (Датчик):** Может расти и убывать.
    - _Пример:_ Использование оперативной памяти, температура CPU, текущее количество активных пользователей.
3. **Histogram (Гистограмма):** Распределяет значения по "корзинам" (buckets). Позволяет считать перцентили.
    - _Пример:_ Время ответа сервера (сколько запросов ответили за <100мс, <500мс и т.д.).
4. **Summary:** Похож на гистограмму, но вычисляет квантили на стороне клиента.
