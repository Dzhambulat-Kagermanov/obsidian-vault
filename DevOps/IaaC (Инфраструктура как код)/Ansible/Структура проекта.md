
Ansible ожидает найти файлы в конкретных папках с конкретными именами. Если вы нарушите структуру, Ansible просто не увидит ваши задачи или переменные.

Эта структура подходит для сложных проектов с ролями и несколькими окружениями. 

```Structure
my_ansible_project/
├── ansible.cfg                 # Конфигурация Ansible
├── requirements.yml            # Зависимости (роли и коллекции из Galaxy)
├── .gitignore                  # Игнорируемые файлы (.vault_pass, *.retry)
├── .vault_pass                 # Пароль vault (НЕ коммитить в git!)
│
├── inventory/                  # Инвентари для разных сред
│   ├── production/
│   │   ├── hosts.yml
│   │   ├── group_vars/
│   │   └── host_vars/
│   └── staging/
│       ├── hosts.yml
│       ├── group_vars/
│       └── host_vars/
│
├── playbooks/                  # Точка входа (плейбуки)
│   ├── site.yml                # Главный плейбук
│   ├── webservers.yml
│   └── dbservers.yml
│
├── roles/                      # Локальные роли проекта
│   ├── common/
│   ├── nginx/
│   └── app/
│
├── collections/                # Локальные коллекции (если есть)
│   └── mycompany/
│
├── plugins/                    # Кастомные плагины
│   ├── filter/
│   ├── lookup/
│   └── callback/
│
├── vars/                       # Глобальные переменные
│   ├── common.yml
│   └── secrets.yml             # Зашифровано через ansible-vault
│
├── files/                      # Статические файлы (для copy)
│   └── configs/
│
├── templates/                  # Jinja2 шаблоны (для template)
│   └── app/
│
└── logs/                       # Логи выполнения (добавить в .gitignore)
```

### Структура для роли:

**Для Роли** обязательная структура выглядит так:

```Structure
role_name/
├── tasks/
│   └── main.yml      # ⚠️ ОБЯЗАТЕЛЬНО: Точка входа для задач
├── handlers/
│   └── main.yml      # (Опционально) Хендлеры
├── vars/
│   └── main.yml      # (Опционально) Переменные с высоким приоритетом
├── defaults/
│   └── main.yml      # (Опционально) Переменные по умолчанию
├── files/            # (Опционально) Статические файлы
├── templates/        # (Опционально) Jinja2 шаблоны
├── meta/
│   └── main.yml      # (Опционально) Зависимости и метаданные
└── README.md
```

_Главное правило:_ Файл `tasks/main.yml` должен существовать всегда, даже если он пустой. Иначе Ansible выдаст ошибку при попытке запустить роль.

Ansible ищет эту роль в следующих местах (по порядку):

1. **Папка `roles/` относительно текущего плейбука.** Если ваш плейбук лежит в `/home/user/project/site.yml`, Ansible посмотрит в `/home/user/project/roles/my_custom_role`. _Это самый частый сценарий для проектов._
2. **Пути, указанные в `ansible.cfg`.** В конфиге может быть строка: `roles_path = ./roles:/usr/share/ansible/roles`. Он проверит все перечисленные пути.
3. **Системная папка по умолчанию.** Обычно это `/etc/ansible/roles`.
4. **Домашняя директория пользователя.** `~/.ansible/roles`.
5. **Установленные коллекции.** Если роль не найдена как отдельная папка, Ansible попробует найти её внутри установленных коллекций (например, `community.general.my_role`).

### Структура для коллекции:

**Для Коллекции** структура еще строже и включает файл манифеста:

```
namespace-collection/
├── galaxy.yml        # ⚠️ ОБЯЗАТЕЛЬНО: Метаданные (имя, версия, автор)
├── README.md
├── roles/            # Папка с ролями (структура как у обычной роли внутри)
├── plugins/          # Папка с модулями/фильтрами
│   ├── modules/
│   └── filter/
└── meta/
    └── runtime.yml   # (Часто обязательно) Указывает зависимости и версии
```
