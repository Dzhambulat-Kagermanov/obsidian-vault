
GRE - это протокол туннелирования сетевого уровня, который инкапсулирует пакеты одного протокола внутри пакетов другого протокола.

**Режимы работы**
* **Point-to-Point (P2P)** — классический туннель между двумя узлами;
* **Multipoint GRE (mGRE)** — один интерфейс может работать с множеством удаленных узлов (используется в DMVPN)

Базовая настройка в linux (временная):
```bash
# Параметры:
# Локальный внешний IP: 192.168.1.10
# Удаленный внешний IP: 203.0.113.50
# IP внутри туннеля: 10.0.0.1/30 (локальный), 10.0.0.2/30 (удаленный)

# 1. Создание туннеля
ip tunnel add gre0 mode gre remote 203.0.113.50 local 192.168.1.10 ttl 255

# 2. Поднятие интерфейса
ip link set gre0 up

# 3. Назначение IP-адреса туннелю
ip addr add 10.0.0.1/30 dev gre0

# 4. Настройка MTU (важно! GRE добавляет 24 байта оверхеда)
ip link set gre0 mtu 1476

# 5. Добавление маршрута через туннель
ip route add 172.16.0.0/24 via 10.0.0.2 dev gre0

# 6. Проверка
ip tunnel show
ip addr show gre0
ping -I gre0 10.0.0.2
```

Настройка в CISCO:
```Router
! Параметры:
! Локальный внешний IP: 192.168.1.10
! Удаленный внешний IP: 203.0.113.50
! IP внутри туннеля: 10.0.0.1/30 (локальный), 10.0.0.2/30 (удаленный)

! Создание туннельного интерфейса
interface Tunnel0
 description GRE Tunnel to Office
 ip address 10.0.0.1 255.255.255.252
 tunnel source 192.168.1.10
 tunnel destination 203.0.113.50
 tunnel mode gre ip
 tunnel key 1234              ! Опционально: ключ для аутентификации
 tunnel path-mtu-discovery    ! Важно: автоматическая настройка MTU
 ip mtu 1476                  ! Ручная настройка MTU
 keepalive 10 3               ! Проверка живости туннеля
!
! Маршрутизация
ip route 172.16.0.0 255.255.0.0 Tunnel0
!
! Firewall (ACL)
access-list 101 permit gre host 192.168.1.10 host 203.0.113.50
access-list 101 deny ip any any log
interface GigabitEthernet0/0
 ip access-group 101 in
```

### Плюсы GRE

| Преимущество                   | Описание                                                                                        |
| ------------------------------ | ----------------------------------------------------------------------------------------------- |
| **Поддержка мультикаста**      | Может передавать multicast-трафик (важно для OSPF, EIGRP, видеостриминга)                       |
| **Протокольная независимость** | Инкапсулирует практически любой протокол: IPv4, IPv6, IPX, AppleTalk, Ethernet                  |
| **Низкий оверхед**             | Всего 24 байта заголовка (20 байт IP + 4 байта GRE) — минимальные потери пропускной способности |
| **Простота настройки**         | Поддерживается практически любым сетевым оборудованием и ОС                                     |
| **Stateless**                  | Не хранит состояние соединения — меньше нагрузки на устройство                                  |
| **Поддержка маршрутизации**    | Идеален для передачи протоколов динамической маршрутизации (OSPF, BGP, EIGRP)                   |
| **Гибкость**                   | Поддерживает ключи, последовательности, checksums для расширенной функциональности              |
### Минусы GRE

|Недостаток|Описание|Последствия|
|---|---|---|
|**❌ Нет шифрования**|Данные передаются в открытом виде|Не использовать через публичный Интернет без дополнительного шифрования|
|**❌ Нет встроенной аутентификации**|Любой может поднять туннель, зная IP-адреса|Уязвим к спуфингу и MITM-атакам|
|**❌ Проблемы с NAT**|Протокол 47 не имеет портов, сложно пробрасывать через NAT|Требует NAT Traversal или обходных решений|
|**❌ Фрагментация**|Увеличение размера пакета может привести к фрагментации|Нужно тщательно настраивать MTU/MSS|
|**❌ Отсутствие контроля целостности**|Нет проверки, что пакет не был изменен в пути|Риск подмены данных|
|**❌ Нет защиты от replay-атак**|Злоумышленник может перехватить и повторно отправить пакеты|Требует дополнительных механизмов защиты|
