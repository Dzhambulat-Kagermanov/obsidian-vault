**IS-IS (Intermediate System to Intermediate System)** — это протокол динамической маршрутизации внутреннего шлюза (IGP), используемый для обмена информацией о маршрутах внутри одной автономной системы (AS).

Главный смысл IS-IS заключается в **масштабируемости** и **стабильности**.
В отличие от многих других протоколов, IS-IS работает не «поверх» IP, а напрямую поверх канального уровня (Layer 2). Это делает его независимым от сетевой связанности (Network Layer). Хотя сегодня он почти всегда используется для IP, эта архитектура позволяет ему легко поддерживать и другие протоколы (например, IPv6 был внедрен в IS-IS раньше и проще, чем в OSPFv2).

**Ключевые особенности:**
*   **Тип алгоритма:** Link-State (состояние каналов), как и OSPF. Каждый роутер строит полную карту сети (LSDB — Link State Database) и самостоятельно вычисляет кратчайшие пути.
*   **Алгоритм вычисления:** Использует алгоритм Дейкстры (SPF — Shortest Path First).
*   **Метрика:** По умолчанию использует стоимость (cost), зависящую от ширины канала. Стандартная метрика — 10 бит (макс. 63 на линк), но в современных реализацияях используются расширенные метрики (wide metrics, 24 бита), позволяющие задавать огромные значения.
### Адресация в СLNP:

В мире IS-IS термины отличаются от привычного мира IP:
*   **End System (ES):** Хост (компьютер, сервер).
*   **Intermediate System (IS):** Маршрутизатор.
*   **NSAP (Network Service Access Point):** Аналог IP-адреса, но для OSI. У каждого роутера есть уникальный системный ID (часть NSAP).
*   **NET (Network Entity Title):** Особый вид NSAP, где селектор ноды равен 0. Это идентифицирует сам роутер в домене маршрутизации.

**Иерархия уровней.** IS-IS делит сеть на два уровня, что критически важно для масштабирования:

1. **Level 1 (L1):** Внутридоменная маршрутизация. Роутеры знают топологию только своей области (Area). Они похожи на обычные роутеры в зоне OSPF. Чтобы выйти за пределы своей области, они отправляют трафик на ближайший L1/L2 роутер (аналог ABR в OSPF);
2. **Level 2 (L2):** Междоменная маршрутизация (Backbone). Роутеры этого уровня строят карту магистральной сети, соединяющей разные области. L2-роутеры не обязаны знать детали топологии внутри каждой L1-области;
3. **Level 1-2:** Роутеры, поддерживающие взаимодействия обоих уровней.

**Процесс работы:**

1. **Hello-пакеты (IIH - IS-IS Hello):** Роутеры рассылают их на канальном уровне (MAC-адреса). Они обнаруживают соседей и договариваются об уровне (L1 или L2).
2. **Обмен LSP (Link State PDU):** Вместо LSА (как в OSPF), IS-IS использует LSP. Роутер рассылает информацию о своих линках всем соседям.
3. **Синхронизация БД:** Роутеры собирают все LSP в базу LSDB.
4. **Расчет путей:** Запускается SPF-алгоритм, строится дерево кратчайших путей, заполняется таблица маршрутизации.

![[isis-net-address.webp|697]]
**NET идентификатор (идентификатор сети):**

- **AFI (Authority and Format Identifier)** - является частью номера области, хотя постоянно изображается отдельно.    Подавляющее большинство реализаций IS-IS на маршрутизаторах имеют это поле, равным 49. Адреса, у которых AFI был равен 49, относились к классу локальных.
- **Area ID**, номер области, к которой принадлежит маршрутизатор. Это поле переменной длины.
- **System ID** - это идентификатор маршрутизатора. У каждого маршрутизатора в топологии он должен быть уникальным,    поскольку именно по System ID маршрутизаторы "узнают" друг друга при осознании топологии.    В OSPF аналогом System ID является Router ID (однако он имеет другой формат).

В идентификатор сети удобно "забивать" например ip адрес loopback интерфейса и номер локации, для облегчения дебага/траблшутинга в виде. Тогда как вариант идентификатор сети будет вида: `49.xxyy.zzzz.zzzz.zzzz.00`
* **xx** - номер локации; 
- **yy** - номер по порядку оборудования;
- **zzzz.zzzz.zzzz** - ip адрес loopback интерфейса - если IP-адрес 199.106.168.253, то zzzz.zzzz.zzzz\=\=1991.0616.8253 - если IP-адрес 199.17.7.233, то zzzz.zzzz.zzzz\=\=1990.1700.7233 (недостающие разряды дополняются нулями).

Пример:
```Router
net 49.0001.1990.1700.7233.00
```
- **00** - локация 0 ;
- **01** - номер по порядку оборудования;
- **1990.1700.7233** - ip адрес loopback-а 199.17.7.233;

### Дизайн сети IS-IS vs OSPF

В отличии от сети OSPF, где маршрутизатор может находиться одновременно в нескольких областях (OSPF AREA), т.к. раздел области идет по интерфейсу:

![[ospf-design.jpg]]

В сети IS-IS разделение происходит областей происходит по маршрутизатором целиком т.е. маршрутизатор не может быть одновременно в разных областях.

![[isis-design.jpg|697]]


> С точки зрения дизайна протокола нижеследующий дизайн не может быть рабочим для IS-IS.

![[isis-design-wrong.jpg|697]]

На изображении показаны **три области IS-IS**:
- **Area 49.0123**: R1, R2, R3 (L1 соседства);
- **Area 49.0589**: R4, R5, R6 (L1 соседства);
- **Area 49.4578**: R7, R8, R9 (L1 соседства)

Между R4 и R6 (которые в одной области 49.0589) установлено **только L1 соседство** (зеленая линия).

В IS-IS **все маршрутизаторы уровня 2 должны образовывать единую непрерывную сеть**. R4 и R6 оба являются L1/L2 роутерами (иначе они не могли бы иметь L2 соседства с R3 и R7), но **между ними нет L2 соседства**!

1. **R3** (L1/L2) имеет L2 соседство с **R4** (L1/L2)
2. **R4** и **R6** имеют только **L1 соседство** (потому что в одной области)
3. **R6** (L1/L2) имеет L2 соседство с **R7** (L1/L2)

**Результат**: L2 LSDB (база данных состояний каналов уровня 2) **не может быть синхронизирована** между R4 и R6, потому что между ними нет L2 adjacency.

#### Как должно быть правильно?


**Вариант 1. Непрерывный L2 backbone**. Между R4 и R6 должно быть L2 соседство (даже если они в одной области!).
