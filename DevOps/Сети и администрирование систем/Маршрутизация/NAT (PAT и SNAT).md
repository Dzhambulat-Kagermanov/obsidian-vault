### Основы:

**NAT (Network Address Translation)** — это технология преобразования сетевых адресов, которая позволяет изменять IP-адреса в заголовках пакетов при их прохождении через маршрутизатор или межсетевой экран.

**Основные цели NAT:**
- Экономия публичных IPv4-адресов;
- Сокрытие внутренней структуры сети;
- Повышение безопасности (внутренние хосты не видны извне);
- Упрощение миграции провайдеров.

**Типы базового NAT:**

| Тип             | Описание                                         | Пример использования                       |
| --------------- | ------------------------------------------------ | ------------------------------------------ |
| **Static NAT**  | Один к одному: внутренний адрес ↔ внешний адрес  | Публикация сервера из внутренней сети      |
| **Dynamic NAT** | Пул публичных адресов распределяется динамически | Корпоративная сеть с ограниченным пулом IP |

### PAT:

**PAT (Port Address Translation)** — это расширенная версия NAT, также известная как **NAT Overload** или **NAPT** (Network Address Port Translation).

#### Процесс преобразования адресов:
```
Внутренняя сеть:          Маршрутизатор:          Интернет:
192.168.1.10:1234  ──►  203.0.113.1:50001  ──►  Сервер в интернете
192.168.1.11:5678  ──►  203.0.113.1:50002  ──►  Сервер в интернете
192.168.1.12:9012  ──►  203.0.113.1:50003  ──►  Сервер в интернете
```

**Ключевые особенности PAT:**
- **Многие к одному**: множество внутренних адресов используют один публичный IP;
- Использует **порты транспортного уровня** (TCP/UDP) для различения сессий;
- Таблица трансляций хранит: `внутренний_IP:порт ↔ внешний_порт`;
- Наиболее распространённый тип NAT в домашних и офисных сетях.

Пример конфигурации:
```Router
interface GigabitEthernet0/0
 ip nat inside
!
interface GigabitEthernet0/1
 ip nat outside
!
ip nat pool <NAME> 192.168.10.1-192.168.10.254
ip nat source dynamic inside-to-ouside pool <NAME> overload interface  GigabitEthernet0/1
```

> Ключевое слово `overload` активирует режим PAT.

### SNAT:

**SNAT** — это преобразование **исходного адреса** пакета. Термин чаще используется в контексте межсетевых экранов (iptables, pfSense, FortiGate) и облачных инфраструктур.

**Когда применяется SNAT:**
- Внутренний хост инициирует соединение с внешним ресурсом
- Необходимо скрыть реальный источник трафика
- Требуется обеспечить возврат трафика через тот же шлюз
