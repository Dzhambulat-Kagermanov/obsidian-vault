### Основы:

**NAT (Network Address Translation)** — это технология преобразования сетевых адресов, которая позволяет изменять IP-адреса в заголовках пакетов при их прохождении через маршрутизатор или межсетевой экран.

**Основные цели NAT:**
- Экономия публичных IPv4-адресов;
- Сокрытие внутренней структуры сети;
- Повышение безопасности (внутренние хосты не видны извне);
- Упрощение миграции провайдеров.

**Типы базового NAT:**

| Тип             | Описание                                         | Пример использования                       |
| --------------- | ------------------------------------------------ | ------------------------------------------ |
| **Static NAT**  | Один к одному: внутренний адрес ↔ внешний адрес  | Публикация сервера из внутренней сети      |
| **Dynamic NAT** | Пул публичных адресов распределяется динамически | Корпоративная сеть с ограниченным пулом IP |

### PAT:

**PAT (Port Address Translation)** — это расширенная версия NAT, также известная как **NAT Overload** или **NAPT** (Network Address Port Translation).

#### Процесс преобразования адресов:
```
Внутренняя сеть:          Маршрутизатор:          Интернет:
192.168.1.10:1234  ──►  203.0.113.1:50001  ──►  Сервер в интернете
192.168.1.11:5678  ──►  203.0.113.1:50002  ──►  Сервер в интернете
192.168.1.12:9012  ──►  203.0.113.1:50003  ──►  Сервер в интернете
```

**Ключевые особенности PAT:**
- **Многие к одному**: множество внутренних адресов используют один публичный IP;
- Использует **порты транспортного уровня** (TCP/UDP) для различения сессий;
- Таблица трансляций хранит: `внутренний_IP:порт ↔ внешний_порт`;
- Наиболее распространённый тип NAT в домашних и офисных сетях.

**Пример конфигурации:**
```Router
interface GigabitEthernet0/0
 ip nat inside
!
interface GigabitEthernet0/1
 ip nat outside
!
ip nat pool <NAME> 192.168.10.1-192.168.10.254
ip nat source dynamic inside-to-ouside pool <NAME> overload interface  GigabitEthernet0/1
```
> Ключевое слово `overload` активирует режим PAT.

```bash
iptables -t nat -A POSTROUTING -o enp7s1 -j MASQUERADE
```
> `-j MASQUERADE` активирует режим PAT.

### SNAT:

**SNAT** — это жёсткое, постоянное соответствие одного внутреннего приватного адреса одному внешнему публичному адресу.
- **1 внутренний IP ↔ 1 внешний IP**;
- Работает **в обе стороны**: инициация возможна как изнутри, так и снаружи;
- Настраивается **вручную**, запись в таблице не истекает по таймауту;
- Идеально для публикации серверов (веб, почта, VPN).

**Когда применяется SNAT:**
- Внутренний хост инициирует соединение с внешним ресурсом;
- Необходимо скрыть реальный источник трафика;
- Требуется обеспечить возврат трафика через тот же шлюз.

**Пример конфигурации:**
```Router
ip nat inside source static 192.168.1.100 203.0.113.10
!
interface GigabitEthernet0/0
 ip nat inside
!
interface GigabitEthernet0/1
 ip nat outside
```
> Ключевое слово `statis` активирует режим SNAT.

```bash
iptables -t nat -A POSTROUTING -s 192.168.1.100 -j SNAT --to-source 203.0.113.10
```
> `-j SNAT` активирует режим SNAT.

### DNAT:

**DNAT (Destination Network Address Translation)** — это механизм трансляции сетевых адресов назначения, используемый в сетях для перенаправления входящих запросов из интернета на внутренние (приватные) IP-адреса локальной сети, часто используется для проброса портов. При поступлении пакета с внешним IP-адресом, шлюз/роутер меняет адрес назначения на локальный (внутренний) адрес устройства и отправляет его дальше.

Обычно DNAT используется для обеспечения доступа к серверу в частной сети из интернета.

**Отличие от SNAT:** DNAT меняет адрес _назначения_ (входящий трафик), а SNAT (Source NAT) меняет адрес _источника_ (исходящий трафик).

Для работы DNAT роутер должен знать, куда вернуть ответ. Обычно это работает автоматически благодаря **Stateful NAT** (таблица состояний соединений), но внутреннему серверу нужно указать шлюзом этот роутер.

**Пример конфигурации.** Проброс порта 80 на внутренний веб-сервер:

```bash
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf

# 1. DNAT: Меняем назначение пакета
iptables -t nat -A PREROUTING -d 203.0.113.1 -p tcp --dport 80 -j DNAT --to-destination 192.168.1.10:8080

# 2. FORWARD: Разрешаем прохождение трафика (фильтрация)
# Без этого правила пакет будет изменён (DNAT), но отброшен фаерволом
iptables -A FORWARD -d 192.168.1.10 -p tcp --dport 8080 -j ACCEPT

# 3. (Опционально) Ограничение по источнику (безопасность)
# Разрешить доступ только с конкретного IP админа
iptables -A FORWARD -s 198.51.100.5 -d 192.168.1.10 -p tcp --dport 8080 -j ACCEPT
```

```Router
! 1. Подготовка интерфейсов
interface GigabitEthernet0/0
 description Inside LAN
 ip address 192.168.1.1 255.255.255.0
 ip nat inside
!
interface GigabitEthernet0/1
 description Outside WAN
 ip address 203.0.113.1 255.255.255.0
 ip nat outside

! 2. Проброс конкретного порта
ip nat inside source static tcp 192.168.1.10 8080 203.0.113.1 80
! Или можно указать интерфейс
ip nat inside source static tcp 192.168.1.10 8080 interface GigabitEthernet0/1 80

! 3. Проброс диапазона портов
ip nat inside source static tcp 192.168.1.10 8000 203.0.113.1 8000
ip nat inside source static tcp 192.168.1.10 8001 203.0.113.1 8001
```
