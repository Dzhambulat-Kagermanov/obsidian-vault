
Главное отличие `rsync` от `cp` или `scp` — **дельта-кодирование** (delta-transfer algorithm).

1. **Сравнение:** Rsync сравнивает исходные и целевые файлы по размеру и времени последней модификации (by default). Если они отличаются, он вычисляет, какие именно блоки данных изменились.
2. **Передача:** Передается только разница (дельта), а не весь файл. Это критично для больших файлов или медленных каналов.
3. **Сжатие:** Данные сжимаются на лету (флаг `-z`).

**Важный нюанс:** Для вычисления дельты rsync должен прочитать _оба_ файла (источник и получатель). Если ты копируешь на новый диск, где файлов еще нет, rsync просто скопирует их целиком. Экономия трафика начинается при повторных запусках (инкрементальная синхронизация).


### Ключевые флаги и их комбинации:

Самая популярная комбинация, которую нужно знать наизусть:

```bash
rsync -avzh --progress --delete /source/ user@host:/dest/
```

#### Основные режимы:

- `-a` (**archive**): Это макро-флаг. Он эквивалентен `-rlptgoD` (но без `-H` и `-A` по умолчанию в старых версиях, хотя в современных часто включает больше).
    - `-r`: рекурсивно (для директорий).
    - `-l`: копировать симлинки как симлинки (а не содержимое, на которое они указывают).
    - `-p`: сохранять права доступа (permissions).
    - `-t`: сохранять время модификации (критично для того, чтобы rsync понимал, что файл не менялся, и не перекачивал его зря).
    - `-g`: сохранять группу.
    - `-o`: сохранять владельца (требует root прав на стороне приема).
    - `-D`: сохранять устройства (device files) и специальные файлы.
- `-v` (**verbose**): Показывает, что происходит. Без него ты не увидишь список файлов.
- `-h` (**human-readable**): Числа в КБ, МБ, ГБ, а не в байтах.
- `-z` (**compress**): Сжимает данные при передаче по сети. _Нюанс:_ Если канал быстрый (LAN 10Gbps+), а CPU слабый, `-z` может замедлить процесс. В локальной сети часто лучше без него.

#### Управление удалением и синхронизацией:

- `--delete`: **Опасный, но важный флаг.** Удаляет в целевой директории файлы, которых нет в источнике. Делает зеркальную копию.
    - _Без этого флага:_ Если ты удалил файл в `/source`, он останется в `/dest`.
    - _С этим флагом:_ `/dest` станет точной копией `/source`.
- `--delete-excluded`: Удаляет даже те файлы в приемнике, которые подпадают под паттерны исключения.
- `--backup` / `-b`: Вместо удаления или перезаписи, старые файлы переименовываются (добавляется суффикс, обычно `~`). Можно задать директорию для бэкапов через `--backup-dir=/path/to/backups`.

#### Исключения и фильтры:

Rsync имеет сложную систему фильтрации. Порядок правил важен!

- `--exclude='pattern'`: Исключить файлы/папки.
    - Пример: `--exclude='.git'`, `--exclude='*.log'`, `--exclude='/tmp/'`.
    - Обратите внимание: слэш в начале паттерна (`/tmp/`) означает корень передаваемой директории. Без слэша (`tmp/`) исключит любую папку с именем tmp в любой вложенности.
- `--include='pattern'`: Включить (используется вместе с exclude для сложных правил).
- `--filter='- .git/'`: Более гибкий синтаксис фильтров.

#### Производительность и сеть

- `-P`: Эквивалент `--partial --progress`.
    - `--partial`: Если передача прервалась, не удалять частично скачанный файл. При следующем запуске докачает с места обрыва.
- `--bwlimit=KBPS`: Ограничение пропускной способности (чтобы не забить канал).
- `-e ssh`: Указывает использовать SSH как транспорт. По умолчанию rsync использует свой демон (порт 873) или rsh (устарело). В 99% случаев ты будешь использовать SSH.
    - Можно передать опции SSH: `-e "ssh -p 2222 -i /path/to/key"`.

#### Проверка целостности

- `--checksum` / `-c`: Заставляет rsync сверять файлы по контрольной сумме (MD5/SHA1), а не по времени/размеру.
    - _Минус:_ Очень медленно, так как нужно читать каждый файл целиком на обеих сторонах. Используй только если подозреваешь, что время модификации сбилось или файлы повреждены.

### Best-practice и нюансы в работе

#### Слеш в конце имеет значение:

 Если нужно скопировать содержимое папки, ставим слэш в конце источника. Если нужно скопировать саму папку — не ставь.

1. `rsync -av /src/dir user@host:/dest/`
    - Создаст директорию `dir` внутри `/dest/`.
    - Итог: `/dest/dir/...files...`
2. `rsync -av /src/dir/ user@host:/dest/`
    - Копирует _содержимое_ `dir` прямо в `/dest/`.
    - Итог: `/dest/...files...`

#### Dry-run:

Флаг `--dry-run` или `-n` показывает, что _будет_ сделано, но ничего не меняет. _Всегда_ проверяй вывод, особенно когда используешь `--delete`.

```bash
rsync -avzn --delete /source/ user@host:/dest/
```

#### Атомарность операций:

При обновлении живых сервисов (например, деплой сайта) нельзя просто писать файлы поверх работающих.

1. Синхронизируй файлы во временную директорию (`/var/www/site_new`).
2. После успешного завершения сделай `mv` (переименование) старой директории в бэкап, а новой — в рабочую. `mv` на одной файловой системе атомарен.

```bash
rsync -avz /build/ user@host:/var/www/site_new/
ssh user@host "mv /var/www/site /var/www/site_old && mv /var/www/site_new /var/www/site"
```

#### Обработка ошибок и exit codes:

Rsync возвращает разные коды выхода.

- `0`: Успех.
- `12`: Ошибка в протоколе rsync.
- `23`: Частичная передача (некоторые файлы не удалось передать, например, из-за прав).
- `24`: Файлы исчезли во время передачи.
- `255`: Ошибка SSH соединения.

В скриптах (Bash/Ansible) всегда проверяй `$?`.

#### Использование `--link-dest` для инкрементальных бэкапов:

Если ты делаешь ежедневные бэкапы, не копируй полные данные каждый раз.

```bash
rsync -av --link-dest=/backups/yesterday /source/ /backups/today/
```

Если файл не изменился с вчерашнего дня, rsync создаст жесткую ссылку (hardlink) на файл во вчерашней папке. Это экономит место на диске (файл физически один), но дает видимость полной копии за каждый день.

### Типичные случаи применения:

#### Деплой сайта (зеркалирование):

```bash
rsync -avzh --delete --exclude='.git' --exclude='.env' ./dist/ user@server:/var/www/html/
```

#### Бэкап домашней директории (инкрементальный):

```bash
rsync -avzh --backup --backup-dir=/backups/incremental/$(date +%F) /home/dzhambulat/ user@backup-server:/backups/latest/
```

#### Копирование между двумя удаленными серверами (через локальную машину):

```bash
rsync -avzh user1@server1:/data/ user2@server2:/data/
```

#### Ежедневный backup:

Скрипт инкрементального backup-а:

```bash
#!/bin/bash

# --- КОНФИГУРАЦИЯ ---
SOURCE_DIR="/var/www/html/"       # Что бэкапим (слэш в конце важен!)
DEST_USER="backupuser"
DEST_HOST="192.168.1.100"         # IP или домен сервера бэкапов
DEST_BASE_DIR="/backups/webserver" # Базовая папка на удаленном сервере

# Имя текущей папки бэкапа (например, daily.0)
CURRENT_BACKUP="daily.0"
# Папка с предыдущим бэкапом для создания hard links
PREVIOUS_BACKUP="daily.1"

# Опции Rsync
RSYNC_OPTS="-avzh --delete --numeric-ids"
# --numeric-ids: не мапить UID/GID по именам (важно, если пользователи разные на машинах)
# --delete: удалять файлы на приемнике, если их нет в источнике (в рамках этого снапшота)

# Исключения
EXCLUDES="--exclude='.git' --exclude='cache/*' --exclude='*.tmp'"

# SSH настройки
SSH_OPTS="-o StrictHostKeyChecking=no -i /home/dzhambulat/.ssh/backup_key"

# --- ЛОГИКА ---

# 1. Проверяем связь
if ! ping -c 1 -W 2 $DEST_HOST &> /dev/null; then
    echo "Error: Host $DEST_HOST is unreachable."
    exit 1
fi

# 2. Формируем полные пути
DEST_CURRENT="${DEST_USER}@${DEST_HOST}:${DEST_BASE_DIR}/${CURRENT_BACKUP}/"
DEST_PREVIOUS="${DEST_USER}@${DEST_HOST}:${DEST_BASE_DIR}/${PREVIOUS_BACKUP}/"

# 3. Запускаем rsync
# Ключевой момент: --link-dest
# Если файл не изменился по сравнению с PREVIOUS_BACKUP, rsync создаст жесткую ссылку на него.
# Если изменился — запишет новый файл.
echo "Starting backup to ${DEST_CURRENT}..."

rsync $RSYNC_OPTS \
    $EXCLUDES \
    -e "ssh $SSH_OPTS" \
    --link-dest=$DEST_PREVIOUS \
    $SOURCE_DIR \
    $DEST_CURRENT

EXIT_CODE=$?

if [ $EXIT_CODE -eq 0 ]; then
    echo "Backup completed successfully."
else
    echo "Backup failed with exit code $EXIT_CODE"
    # Здесь можно добавить отправку уведомления в Telegram/Email
fi

exit $EXIT_CODE
```

Делаем файл исполняемым:

```bash
chmod +x /usr/local/bin/backup_rsync.sh
```

Планирование запуска каждый день в 3 ночи:

```bash
crontab -e
```

```cron
0 3 * * * /usr/local/bin/backup_rsync.sh >> /var/log/rsync_backup.log 2>&1
```
